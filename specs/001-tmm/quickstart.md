# Quickstart: Time Management Matrix (TMM)

**Feature**: [spec.md](./spec.md) | **Plan**: [plan.md](./plan.md)

---

## What is the TMM?

A standalone HTML tool that renders a four-quadrant Impact × Urgency canvas. Users place their Big Rocks (initiatives) as draggable dots, assign categories, add details, and save/load to a shared JSON project file.

## Architecture in 30 Seconds

```text
public/tools/tmm/index.html   ← Single file, all JS/CSS inline
         │
         ├── CSS: Custom properties, grid layout, dot styling
         ├── HTML: Toolbar, canvas (4 quadrants), detail panel, add-initiative form
         └── JS: State management, Pointer Events drag, File API save/load
                    │
                    └── Shared project file (effectivenessToolkit envelope)
                        └── initiatives[].tmm = { category, details, x, y }
```

## Key Design Decisions

| Decision | Choice | Why |
|----------|--------|-----|
| Drag API | Pointer Events | Smooth real-time tracking, unified mouse/touch, `setPointerCapture` |
| Canvas rendering | CSS divs + absolute positioning | Simplest for ~15 elements; native DOM events; accessible |
| Data placement | `tmm` section per initiative (shared array) | Follows COER pattern; Constitution Principle VI |
| Label truncation | CSS `text-overflow: ellipsis` at 30 chars | Zero-effort, full name in detail panel |

## Coordinate System

```text
Impact (y=1.0)
  ┌──────────────┬──────────────┐
  │ Q1 Fire-     │ Q2 Growth    │   top = (1 - y) * 100%
  │ fighting     │              │   left = x * 100%
  ├──────────────┼──────────────┤
  │ Q3 Distract. │ Q4 Waste     │
  └──────────────┴──────────────┘
(x=0.0)                (x=1.0)
More Urgent            Less Urgent
```

- `x` = normalised Urgency (0.0 = high/left, 1.0 = low/right)
- `y` = normalised Impact (0.0 = low/bottom, 1.0 = high/top)
- CSS mapping: `left = x * 100%`, `top = (1 - y) * 100%`
